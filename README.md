
```# Документация Telegram-бота для управления сервером (DOCS.MD)

Данный бот разработан на фреймворке **Aiogram 3.x** и предназначен для удаленного администрирования, мониторинга состояния и управления файлами/процессами Linux-сервера через интерфейс Telegram.


## 🛠 1. Требования к системе и окружению

Для корректной работы всех модулей на целевом сервере должны быть выполнены следующие условия:

1. **Python 3.10+** и установленные зависимости (`aiogram`, `psutil`, `python-dotenv`, `pymongo` и др.).
2. **Lm-sensors** — утилита для мониторинга температуры процессора и скорости кулеров, используемая в модуле `status.py`. 
   * *Установка в Ubuntu/Debian:* `sudo apt update && sudo apt install lm-sensors && sudo sensors-detect`
3. Настроенные привилегии **Sudo** без ввода пароля для системных команд, вызываемых ботом.


## 🔑 2. Важная настройка Прав Доступа (sudo visudo)

Поскольку бот работает от лица ограниченного системного пользователя, но выполняет административные действия (удаление файлов/папок, завершение процессов других пользователей), необходимо выдать ему точечные права `sudo` без запроса пароля.

Выполните команду для редактирования файла конфигурации sudoers:
```bash
sudo visudo

```

Добавьте в конец файла следующую строку (замените `your_bot_user` на реальное имя пользователя, под которым запускается бот):

Plaintext

```
your_bot_user ALL=(ALL) NOPASSWD: /usr/bin/kill, /bin/kill, /usr/bin/rm, /bin/rm

```

### Зачем это нужно?

-   **Диспетчер задач (`task_manager.py`):** Использует консольные команды `sudo kill <PID>` и `sudo kill -9 <PID>` для штатного или принудительного уничтожения выбранных процессов. Без настройки `visudo` вызовы завершатся ошибкой доступа.
    
-   **Файловый менеджер (`file_manager.py`):** При удалении директорий (`shutil.rmtree`) или защищенных файлов боту могут потребоваться повышенные привилегии, если операции затрагивают папки вне домашней директории.
    

## 📂 3. Структура проекта

-   `bot.py` — Главная точка входа. Инициализирует бота, динамически загружает роутеры и подключает middleware.
    
-   `config.py` — Файл конфигурации. Считывает переменные окружения, задает списки разрешенных (`ALLOWED_DIRS`) и опасных директорий.
    
-   `.env` — Конфиденциальные данные (токен бота, ID администраторов).
    
-   `routes/` — Каталог с модулями функционала (роутерами):
    
    -   `start.py` — Приветственное сообщение и главное меню.
        
    -   `status.py` — Сбор метрик сервера (CPU, RAM, Диски, Температура через `sensors`).
        
    -   `task_manager.py` — Мониторинг процессов и управление ими (терминация, перезапуск).
        
    -   `file_manager.py` — Навигация по серверу, скачивание, редактирование, создание и удаление файлов/папок.
        
    -   `commands.py` — Прямое выполнение терминальных команд на сервере.
        
    -   `callbacks.py` — Общие обработчики (например, кнопка отмены).
        
-   `middleware/middleware.py` — Фильтрация входящих апдейтов (доступ разрешен только пользователям из `ADMIN_IDS`).
    
-   `utils/` — Вспомогательные скрипты (`logger.py`, `pc.py` для парсинга системных данных, `command.py` для безопасного выполнения).
    

## ⚙️ 4. Конфигурация (.env)

Перед запуском создайте файл `.env` в корневой директории:

Фрагмент коду

```
# Токен бота от @BotFather
TELEGRAM_BOT_TOKEN=YOUR_TELEGRAM_BOT_TOKEN

# ID администраторов через запятую, у которых будет доступ к боту
ADMIN_IDS=123456789,987654321

# Уровень логирования
LOG_LEVEL=INFO


```

## 🚀 5. Функционал и Команды

Доступ к боту строго ограничен с помощью `BotMiddleware`. Если Telegram ID пользователя отсутствует в `ADMIN_IDS`, бот полностью игнорирует сообщения.

### 📊 Статистика сервера

-   **Триггер:** Кнопка `📊 Статистика сервера`
    
-   **Описание:** Собирает текущие данные о системе:
    
    -   Название процессора, нагрузка и температура (требуется `lm-sensors`).
        
    -   Использование оперативной памяти (RAM) в GB и процентах.
        
    -   Информация по подключенным дискам (точка монтирования, свободное/занятое пространство).
        

### ⚡️ Диспетчер задач

-   **Триггер:** Кнопка `⚡️ Диспетчер задач`
    
-   **Описание:** Выводит интерактивный список процессов с пагинацией (по 10 штук), отсортированных по потреблению RAM. При клике на процесс доступны действия:
    
    -   **Детали:** Просмотр PID, CPU%, путя к файлу запуска.
        
    -   **SIGTERM:** Мягкая остановка процесса через `sudo kill`.
        
    -   **SIGKILL:** Принудительное уничтожение через `sudo kill -9`.
        
    -   **Перезапуск:** Бот считывает `cmdline` и `cwd`, убивает процесс и запускает его заново в фоне.
        

### 📁 Файловый менеджер

-   **Триггер:** Кнопка `📁 Файловый менеджер`
    
-   **Описание:** Начиная с пути, указанного в `ALLOWED_DIRS[0]`, позволяет полноценно работать с файловой системой:
    
    -   Перемещение по папкам (вперед/назад).
        
    -   Скачивание файлов в Telegram (размером до 500 МБ).
        
    -   Создание новых файлов и папок, переименование объектов.
        
    -   Удаление файлов и рекурсивное удаление папок.
        
    -   Редактирование текстовых файлов прямо в чате (до 1 МБ).
        

### ⌨️ Выполнить команду

-   **Триггер:** Кнопка `⌨️ Выполнить команду`
    
-   **Описание:** Переводит бота в состояние ожидания текстовой команды. Полученная строка отправляется на выполнение в shell-оболочку сервера через `SecureCommandExecutor`. Возвращает статус выполнения и стандартный вывод (stdout/stderr).
    

## 🏃 6. Запуск бота

Для запуска в ручном режиме выполните:

Bash

```
python bot.py

```

Для обеспечения бесперебойной работы в продакшене рекомендуется настроить **systemd** службу:

1.  Создайте файл службы `/etc/systemd/system/tgbot.service`:
    

Ini, TOML

```
[Unit]
Description=Telegram Server Administration Bot
After=network.target

[Service]
Type=simple
User=your_bot_user
WorkingDirectory=/path/to/your/bot
ExecStart=/usr/bin/python3 bot.py
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target

```

2.  Перезапустите демон systemd, включите и запустите службу:
    

Bash

```
sudo systemctl daemon-reload
sudo systemctl enable tgbot.service
sudo systemctl start tgbot.service
```
